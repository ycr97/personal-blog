# 工厂模式

工厂模式是一种创建型设计模式，它提供了一种创建对象的方式，将创建对象的实例的过程从主代码中抽离出来，以提高代码的可读性和可维护性。

### 简单工厂

以下是一个简单工厂的示例

定义一个Animal接口以及两个实现类Cat和Dog

```java
public interface Animal {
    void speak();
}
```

```java
public class Cat implements Animal{
    @Override
    public void speak() {
        System.out.println("喵喵喵!");
    }
}
```

```java
public class Dog implements Animal{
    @Override
    public void speak() {
        System.out.println("汪汪汪!");
    }
}
```

定义一个AnimalFactory提供一个createAnimal的静态方法可以通过传入的类型来创建不同的Animal类型的实例

```java
public class AnimalFactory  {

    public static Animal createAnimal(String type) {
        Animal animal = null;
        switch (type) {
            case "dog":
                animal = new Dog();
                break;
            case "cat":
                animal = new Cat();
                break;
            default:
                break;
        }
        return animal;
    }
}
```

最后我们可以尝试使用AnimalFactory来创建不同的对象

```java
public class SimpleFactoryMain {

    public static void main(String[] args) {
        Animal cat = AnimalFactory.createAnimal("cat");
        cat.speak();

        Animal dog = AnimalFactory.createAnimal("dog");
        dog.speak();
    }
}
```

运行结果

```java
喵喵喵!
汪汪汪!
```
# 实际运用

最近做一个功能, 从三方公司存的双录质检视频下载并上传到我们公司的文件存储服务；需要将视频从OSS服务器下载,并且将视频上传到我们的OSS服务。其中视频的链接和视频的一些信息比如视频大小、质检时间、质检员等等信息将有三方公司提供接口供我们获取。并且需要对接多个经代公司, 并且后面还会新增新的经代公司。其中下载视频和上传至我们的文件服务的功能可以封装为公用逻辑,但是调用三方公司的接口的逻辑是各不相同的。比方说路径, 报文加解密方式, 返回字段各有不同所以需要对不同的经代公司获取视频信息有不同处理方式。在开发这个功能的时候考虑到代码设计的开闭原则, 对新增是开放的, 对修改是关闭的。此时如果针对不同的经代公司获取视频信息采用简单的if判断或者switch来处理,那新增一个经代公司就需要在处理类中加一个判断, 并且不断加逻辑在一个类里面。违背了开闭原则而且不利于代码的维护。所以采用策略模式加工厂模式的方式，针对去请求不同的接口对应不同的获取视频信息的类, 所有获取视频信息的处理类统一实现接口或者继承抽象类。 由发起方传的标识获取对应的处理类。这样就避免了会不断的在一个类里面添加新的逻辑, 新增一个经代公司就新增一个对应公司的处理类,利于维护我们的代码。为了避免在主处理类中不断添加if的逻辑我们采用自动获取处理类, 将所有的处理类加载到一个Map中由工厂根据经代的渠道标识来获取。初始化时就会将所有的处理类加载到Map中, Map的key对应渠道标识, Map的value对应处理类。结合Spring的自动注入实现这个功能会变得十分简单，因为将所有的处理类注入到Map由Spring框架自动实现。

以下是代码示例: 

视频处理抽象类

```java
/**

public abstract class AbstractVideoProcessor {

    abstract void videoProcess();
```

```java

    }
}
```


视频处理类工厂

```java
/**
 * @author ycr
 * @date 2024/6/23 14:31
 */
@Component
public class VideoProcessorFactory {
    public static final String SUFFIX = "_CompanyVideoProcessor";

    private final Map<String, AbstractVideoProcessor> processorMap;

    @Autowired
    public VideoProcessorFactory(Map<String, AbstractVideoProcessor> processorMap) {
        this.processorMap = processorMap;
    }

    public AbstractVideoProcessor getVideoProcessor(String processorName) {
        // Assert.isNull(processorName, "processorName is null");
        return processorMap.get(processorName + SUFFIX);
    }
}

```

业务类

```java
/**
 * @author ycr
 * @date 2024/6/23 14:47
 */
@Service
public class VideoProcessServiceImpl {

    @Autowired
    private VideoProcessorFactory videoProcessorFactory;

    public void videoProcess(String companyCode) {
        AbstractVideoProcessor videoProcessor = videoProcessorFactory.getVideoProcessor(companyCode);
        videoProcessor.videoProcess();
    }
}

```
